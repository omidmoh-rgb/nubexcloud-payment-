Integration Guide

HubSpot CRM Setup
for Nubexcloud Payments

Follow these steps once. Every confirmed payment will automatically appear as a Contact + Deal in your HubSpot account — completely invisible to the customer.

How it works
N-GeniusProcesses payment
→
Railway ServerConfirms status
→
HubSpot CRMContact + Deal created
ℹ️
The push to HubSpot happens entirely on the server when the payment status is confirmed as CAPTURED or AUTHORISED. The customer sees only the normal success page — no delay, no awareness.
1
HubSpot

Log in to your HubSpot account

Go to app.hubspot.com and sign in with your admin account. You need a Super Admin or account with Private App permissions.

2
Navigation

Open Private Apps settings

Click your account name (top-right) → Settings. Then in the left sidebar go to:

⚙ Settings
›
Integrations
›
Private Apps

Click the orange Create a private app button.

3
App Info

Name and describe the app

Fill in these fields
App nameNubex Payment Integration
DescriptionAuto-records N-Genius payment transactions
4
Permissions — Critical

Grant exactly these scopes

Click the Scopes tab. Search for and enable each of the following. Get this right — too few and the integration will fail.

Required scopes
crm.objects.contacts.read
crm.objects.contacts.write
crm.objects.deals.read
crm.objects.deals.write
crm.schemas.deals.read
read = read write = create / update
⚠️
Do not grant more permissions than listed above. Least-privilege keeps your HubSpot data safe if the token is ever exposed.
5
Token

Create the app and copy your token

Click Create app. HubSpot will show a Private App Token — a long string starting with pat-na1-…

⚠️
Copy it immediately. HubSpot only shows it once. If you lose it, you must rotate (delete + recreate) the token.
6
Railway

Add the token to Railway environment variables

Go to railway.app → your project → Variables tab. Add a new variable:

.env / Railway variable
HUBSPOT_ACCESS_TOKEN=pat-na1-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
💡
Railway automatically redeploys when you save a new variable. No extra steps needed — the server picks it up on next boot.
7
Deploy

Replace server.js on GitHub

Push the new server.js file to your GitHub repo. Railway will auto-deploy. Verify the startup log shows:

Railway logs
║  Nubexcloud Payment Server — LIVE        ║
  Port    : 3000
  Mode    : 🟢 LIVE
  HubSpot : ✓ Set        ← must show this
⚠️
If it shows ⚠ NOT SET the variable wasn't saved correctly. Go back to Railway → Variables and check the name is exactly HUBSPOT_ACCESS_TOKEN.
8
Verify

Test with a sandbox payment

Make a test payment using your sandbox mode. Then open HubSpot → CRM → Contacts and CRM → Deals. You should see:

👤 Contact created
Emailcustomer-{ref}@nubexcloud.com
First nameCustomer
Last name{order reference}
LifecycleCustomer
💰 Deal created
NameNubex Payment — {ref}
Amount{USD / AED / EUR}
StageClosed Won
GatewayN-Genius
✅
If you see both records — the integration is live. Every future payment, in any currency (USD, AED, EUR), will automatically create a Contact + Deal in your HubSpot.
✓
Health Check

Expected JSON response

  "env"       : "live",
  "configured": true,
